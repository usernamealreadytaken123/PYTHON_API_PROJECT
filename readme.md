# Weather API

## Описание

Данный проект представляет собой HTTP-сервер на Python для предоставления информации о погоде.
Сервер реализован с использованием асинхронного фреймворка **FastAPI**.
Данные о погоде запрашиваются из внешнего сервиса **Open-Meteo API**.

Приложение предоставляет REST API для выполнения следующих задач:

1. Получение текущей погоды по координатам.
2. Добавление города в список отслеживаемых с сохранением прогноза на текущий день.
3. Получение списка городов, для которых доступен сохранённый прогноз.
4. Получение погоды для выбранного города на указанное время с возможностью выбрать нужные погодные параметры.

Также в приложении реализовано **фоновое обновление прогнозов каждые 15 минут**.

Основной файл приложения: `script.py`

## Хранение данных

Данные сохраняются в файле `storage.json`.

В файле хранятся:

* список добавленных городов;
* координаты городов;
* прогноз погоды на текущий день по часам для каждого города;
* время последнего обновления прогноза.

Такой способ хранения соответствует требованиям задания, так как по условию допускается хранение данных в файле.

---

## Логика обновления прогнозов

После запуска приложения создаётся фоновая асинхронная задача, которая:

1. считывает список добавленных городов;
2. заново запрашивает для них прогноз погоды;
3. обновляет сохранённые данные в `storage.json`.

Обновление выполняется **каждые 15 минут**.

## Описание методов API

## 1. Получение текущей погоды по координатам

**Метод:** `GET`
**Endpoint:** `/weather/current`

### Назначение

Метод принимает координаты и возвращает текущие погодные данные для указанной точки.

Возвращаются следующие параметры:

* температура;
* скорость ветра;
* атмосферное давление.

### Параметры запроса

Параметры передаются как query-параметры:

* `lat` — широта
  Допустимый диапазон: от `-90` до `90`

* `lon` — долгота
  Допустимый диапазон: от `-180` до `180`

### Пример запроса

```http
GET /weather/current?lat=55.7558&lon=37.6173
```

Полный пример:

`http://127.0.0.1:8000/weather/current?lat=55.7558&lon=37.6173`

### Пример ответа

```json
{
  "latitude": 55.7558,
  "longitude": 37.6173,
  "timezone": "Europe/Moscow",
  "time": "2026-02-28T18:00",
  "temperature": -5.3,
  "wind_speed": 6.4,
  "pressure": 998.8
}
```

### Возможные ошибки

* `400 Bad Request` — если координаты находятся вне допустимого диапазона;
* `502 Bad Gateway` — если внешний сервис Open-Meteo вернул ошибку или некорректный ответ;
* `504 Gateway Timeout` — если запрос к Open-Meteo не успел выполниться за отведённое время.

## 2. Добавление города в список отслеживаемых

**Метод:** `POST`
**Endpoint:** `/cities`

### Назначение

Метод принимает название города и его координаты, после чего:

1. запрашивает прогноз погоды на текущий день;
2. сохраняет город в список отслеживаемых;
3. сохраняет прогноз в файл `storage.json`.

После успешного выполнения метода для города становится доступен кэш прогноза.

### Тело запроса

Тело запроса передаётся в формате JSON.

Поля:

* `name` — название города;
* `latitude` — широта;
* `longitude` — долгота.

### Пример запроса

```http
POST /cities
Content-Type: application/json

{
  "name": "Moscow",
  "latitude": 55.7558,
  "longitude": 37.6173
}
```

### Пример ответа

```json
{
  "message": "City added and forecast cached",
  "city": "Moscow",
  "forecast_date": "2026-02-28"
}
```

### Возможные ошибки

* `400 Bad Request` — если координаты некорректны или имя города пустое;
* `422 Unprocessable Entity` — если тело запроса не соответствует ожидаемой JSON-структуре;
* `502 Bad Gateway` — если Open-Meteo вернул ошибку или некорректный ответ;
* `504 Gateway Timeout` — если запрос к Open-Meteo не был выполнен вовремя.

### Примечание

Этот метод нельзя вызвать через обычную адресную строку браузера, так как это **POST-запрос** с JSON-телом.

Для вызова метода можно использовать:

* Swagger UI (`/docs`);
* файл `test_main.http` в PyCharm;
* Postman;
* curl.

## 3. Получение списка городов, для которых доступен прогноз

**Метод:** `GET`
**Endpoint:** `/cities`

### Назначение

Метод возвращает список городов, для которых в кэше уже сохранён прогноз.

Для каждого города возвращаются:

* название;
* широта;
* долгота.

### Пример запроса

```http
GET /cities
```

Полный пример:

`http://127.0.0.1:8000/cities`

### Пример ответа

```json
[
  {
    "name": "Moscow",
    "latitude": 55.7558,
    "longitude": 37.6173
  },
  {
    "name": "Seoul",
    "latitude": 37.5665,
    "longitude": 126.978
  }
]
```

### Возможные ошибки

Как правило, метод возвращает `200 OK`.

Если в хранилище ещё нет ни одного города, возвращается пустой список:

```json
[]
```

## 4. Получение погоды для города на указанное время

**Метод:** `GET`
**Endpoint:** `/cities/{city_name}/weather`

### Назначение

Метод возвращает погодные данные для выбранного города на текущий день в указанное время.

Пользователь может:

* получить все доступные погодные параметры;
* выбрать только часть параметров через query-параметр `fields`.

### Параметры запроса

#### Параметр пути

* `city_name` — название города

#### Query-параметры

* `time` — время в формате `HH:MM`
  Пример: `18:00`

* `fields` — необязательный параметр, список нужных полей через запятую

### Доступные значения `fields`

Можно запрашивать следующие поля:

* `temperature` — температура
* `humidity` — влажность
* `wind_speed` — скорость ветра
* `precipitation` — осадки

Если параметр `fields` не указан, возвращаются **все доступные параметры**.

### Пример запроса (все поля)

```http
GET /cities/Moscow/weather?time=18:00
```

Полный пример:

`http://127.0.0.1:8000/cities/Moscow/weather?time=18:00`

### Пример ответа (все поля)

```json
{
  "city": "Moscow",
  "date": "2026-02-28",
  "time": "18:00",
  "data": {
    "temperature": -3.4,
    "humidity": 82,
    "wind_speed": 5.7,
    "precipitation": 0.0
  }
}
```

### Пример запроса (выбор отдельных полей)

```http
GET /cities/Moscow/weather?time=18:00&fields=temperature,wind_speed
```

Полный пример:

`http://127.0.0.1:8000/cities/Moscow/weather?time=18:00&fields=temperature,wind_speed`

### Пример ответа (выбор отдельных полей)

```json
{
  "city": "Moscow",
  "date": "2026-02-28",
  "time": "18:00",
  "data": {
    "temperature": -3.4,
    "wind_speed": 5.7
  }
}
```

### Возможные ошибки

* `400 Bad Request` — если:

  * имя города пустое;
  * параметр `time` не соответствует формату `HH:MM`;
  * в параметре `fields` указаны недопустимые поля;

* `404 Not Found` — если:

  * прогноз для указанного города отсутствует;
  * для указанного времени нет данных в кэше;

* `500 Internal Server Error` — если в сохранённом прогнозе отсутствует дата.

## Особенности реализации

* Приложение реализовано в виде одного исполняемого файла `script.py`.
* Используется асинхронная работа с внешним API через `httpx.AsyncClient`.
* Данные кэшируются локально в файле `storage.json`.
* Для синхронизации операций записи в файл используется `asyncio.Lock`.
* Приложение не требует установки и настройки отдельной базы данных.
* После запуска автоматически создаётся файл `storage.json`, если он отсутствует.
